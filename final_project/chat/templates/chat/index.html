    Slide 1 of 5

    What happens when you ask a question

    Every request goes through a multi-agent pipeline — with guardrails up front and visible progress while it runs.

    1. You ask in the chat or click a pollinator fun-fact card.
    2. A guardrail check confirms the question is about California native plants or pollinators.
    3. The orchestrator routes to the plant expert, pollinator expert, web fun fact agent, or both.
    4. Plant and pollinator experts use RAG over CSV data; the fun fact agent searches the live web with OpenAI's web_search tool.
    5. While agents work, a Working in the background panel shows live progress.
    6. The answer appears in chat with its route label and a collapsible Background steps trace.

    Slide 2 of 5

    The four expert agents

    All agents share the same guardrails and a warm, garden-buddy tone — factual, friendly, and lightly playful.

    Plant retriever

    Landscaping & plant traits

    RAG over plants.csv. Answers only from retrieved records; includes scientific and common names, height, and spread.

    Pollinator expert

    Wildlife & pollinator support

    RAG over pollinator CSVs. Recommends plants by pollinator value — bees, butterflies, hummingbirds, monarchs.

    Fun fact agent

    OpenAI web_search · one sentence

    The only agent that skips RAG and Chroma. It calls the OpenAI Responses API with the hosted web_search tool — search and answer in one step.

    Both + synthesizer

    Combined answer

    When a question needs plant details and pollinator info, both RAG agents run and a synthesizer merges their answers.

    Slide 3 of 5

    OpenAI web_search fun fact agent

    For trivia, this agent skips CSV data and uses OpenAI's hosted web_search tool — search and answer in one step.

    1. Triggered by the router or a pollinator card click.
    2. OpenAI Responses API runs gpt-4o-mini with web_search and returns one playful sentence.
    3. The trace shows the web search step; replies are labeled fun fact in chat.

    RAG datasets & orchestration

    plants.csv

    463 California native plant records with scientific name, common name, height, spread, and description.

    Stored in Chroma at chroma_data/plants_csv

    Polinators_1.csv + polinators_2.csv

    205 pollinator-friendly plant records with pollinator types, plant type, height, and width.

    Stored in Chroma at chroma_data/pollinators_csv

    Router prompt picks one route: plant, pollinator, fun_fact, or both.

    plant and pollinator use Chroma RAG; fun_fact uses OpenAI web_search instead of any vector store.

    Fun-fact cards bypass the router and call the web search agent directly with forced_route="fun_fact".

    The orchestrator returns a trace list — every step from guardrail check through final answer — for the UI to display.

    Guardrails & traceability

    Scope guardrails

    California natives only

    Before any agent runs, GPT classifies the question as in_scope or out_of_scope.

    In scope: native plants, pollinators, garden use, and related fun facts. Out of scope: general trivia, coding, travel, and unrelated topics.

    Blocked questions get a friendly rejection — still warm, still on-brand.

    Background trace

    See what is happening

    While waiting, a spinner panel lists steps like guardrail check, routing, and data search.

    When the answer arrives, expand Background steps on the message to see the full orchestrator trace — guardrail result, route chosen, Chroma CSV lookup or OpenAI web_search, and when the response was ready.

    Tone rules apply to every agent: informal and warm like a knowledgeable garden buddy, factual (no invented plant details), with gentle humor when it fits — never snarky or over-the-top.
